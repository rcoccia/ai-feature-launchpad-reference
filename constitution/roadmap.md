# Roadmap: AI Feature Launchpad

The roadmap contains the confirmed product capability and its human-authorized fresh-context replay.
The replay is didactic evidence for reproducibility, not a second product capability or speculative
product backlog.

---

## Phase 1: Govern one AI-feature launch request

**Goal:** Deliver the single observable launch-approval capability and its complete intent-to-merge
evidence as a public, didactic reference.

- [ ] An internal product team can submit and govern one AI-feature launch request from Pending to an approved or terminally rejected outcome, with distinct requester/Product/AI-Risk actors, durable SQLite state, append-only audit evidence for successful and refused attempts, product-native integration-test CI, and layered public documentation.

---

## Phase 2: Reproduce inception and feature from a fresh context

**Goal:** Prove that an uninvolved fresh agent can reconstruct THIN inception and the same launch-
approval feature from an empty disposable local repository using only the public documentation.

- [ ] Starting with no parent history and without `ask_user`, an uninvolved fresh agent reaches review-ready with all required gates green, records steps, artifacts, latency, rework, defects, and friction, and needs no second cloud repository or merge because the main reference supplies the GitHub merge proof.
